# Roadmap

## 目的

本文件描述 Lemon Platform 的階段性推進方向。Roadmap 用於對齊專案節奏，不代表所有內容都已排定實作。

## Phase 1：文件與基準建立

目標是建立 Project Pack，補齊專案背景、架構、模組、AI 規則與任務模板。

此階段的完成標準是新成員能透過文件理解平台方向，維護者能知道哪些內容屬於本 repo，AI Agent 能依規則協助後續工作。

## Phase 2：既有系統盤點

目標是盤點 Lemon 相關 repository、工具入口、資料來源、營運流程與目前使用中的功能。

此階段需要確認哪些系統已穩定、哪些流程仍依賴人工、哪些資料涉及敏感權限。

## Phase 3：平台入口設計

目標是設計 Lemon Platform 的主入口、中心導覽、首頁摘要與共用 UI 版面。

此階段應以 `tool-system` UI 基準為依據，避免建立不符合既有使用習慣的介面。

## Phase 4：中心功能整合

目標是逐步把 Service、Operations、Finance、Tools、Automation 五大中心整理成可使用的流程。

整合順序應依營運價值、風險與資料成熟度決定，不應只依功能看起來容易與否排序。

## Phase 5：治理與自動化

目標是建立自動化任務管理、執行紀錄、錯誤處理、AI 協作邊界與維護流程。

此階段完成後，平台應能更穩定地支援日常營運與後續擴充。

## 調整規則

Roadmap 需隨實際營運優先順序調整。若某一階段被跳過，應在文件中記錄原因與風險。
