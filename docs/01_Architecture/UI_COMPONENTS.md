# UI 元件規劃

## 目的

本文件定義 Lemon Platform 初期可共用的 UI 元件方向，讓各中心在建立頁面時維持一致的操作語言。

此文件不是完整 Design System，而是後續整理元件規格的起點。

## 導覽元件

導覽元件用於幫助使用者在平台、中心與功能之間切換。側邊導覽應放置主要中心，中心內再依任務區分頁面。

導覽文字應使用營運人員熟悉的詞彙，例如「訂單查詢」、「待付款」、「薪資檢查」、「異動紀錄」，避免使用只有工程端理解的技術名稱。

## 資料元件

資料元件包括表格、摘要卡、狀態標籤、詳細資料區與時間軸。所有資料元件都應讓使用者看得出資料來源與狀態。

表格需要支援排序、篩選、匯出與空資料狀態。摘要卡應只放真正重要的數字，不應用來堆疊所有資訊。

## 操作元件

操作元件包括按鈕、切換、選單、表單欄位、上傳入口與確認動作。操作文字應清楚描述結果，例如「匯出待付款名單」比「送出」更適合。

對資料有影響的操作應提供確認、結果回饋與錯誤訊息。不可逆或高風險操作需要額外提示。

## 回饋元件

回饋元件包含成功訊息、警告、錯誤、載入狀態與執行紀錄。使用者應能知道系統正在做什麼、是否完成、失敗時該如何處理。

自動化任務尤其需要清楚顯示執行時間、輸入來源、輸出位置與錯誤原因。

## 內容語氣

介面文字應短、清楚、可執行。避免使用模糊詞彙，例如「處理中」應盡可能補上正在處理的項目或預期下一步。

錯誤訊息應說明問題與修正方式，不應只顯示技術錯誤。

## 後續整理

後續可將常用元件整理成元件清單，包含名稱、用途、狀態、使用範例與不建議用法。
