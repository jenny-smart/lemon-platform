# Tools Center

## 模組定位

Tools Center 負責集中既有工具、查詢頁、輔助流程與尚未升級為正式中心功能的工作。它是平台吸收既有能力與測試新流程的緩衝區。

## 主要使用者

主要使用者包含營運人員、維護者與需要執行特定工具的人員。使用者通常已有明確任務，例如查名單、整理資料、轉換格式或產出報表。

## 核心任務

Tools Center 初期應支援：

- 集中連結既有工具。
- 整理常用資料查詢。
- 收納一次性或低頻輔助工具。
- 提供工具說明、輸入格式與輸出位置。
- 評估哪些工具可升級為正式模組。

## 工具治理

每個工具都應記錄用途、資料來源、輸入格式、輸出結果、負責人與風險等級。沒有文件的工具不應直接成為平台核心流程。

若工具處理敏感資料，需標示權限與保存規則。

## 與其他中心關係

Tools Center 可以支援所有中心，但不應取代中心責任。當某個工具變成高頻且穩定的業務流程時，應移入對應中心並建立正式規格。

## 待確認事項

後續需盤點既有 `tool-system` 工具、確認仍在使用的功能、移除過期入口，並建立工具清單。
