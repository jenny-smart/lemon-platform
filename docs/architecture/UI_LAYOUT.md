# UI 版面規劃

## 版面目標

Lemon Platform 的版面應讓使用者快速辨識工作中心、目前頁面與可執行任務。平台不是行銷網站，畫面設計應以工作效率、資訊密度與清楚導覽為優先。

## 基本結構

標準頁面建議包含下列區域：

- 頁首區：顯示中心名稱、頁面標題與必要狀態。
- 側邊導覽：顯示中心與功能入口。
- 內容區：呈現主要資料、篩選條件、操作結果與表格。
- 輔助區：顯示提示、警告、最近紀錄或輸出說明。

不同中心可以調整內容，但不應改變使用者對平台結構的基本預期。

## 首頁布局

平台首頁應提供中心入口、待處理摘要與常用任務。首頁不應變成公告牆，也不應堆疊過多低頻資訊。

建議首頁內容包括：

- 今日或近期需處理事項。
- 五大中心入口。
- 常用工具捷徑。
- 系統狀態或資料同步提醒。

## 中心頁布局

每個中心首頁應以任務導向安排內容。例如 Finance Center 應突出付款、對帳、薪資與費用；Operations Center 應突出派工、異動、服務狀態與營運追蹤。

中心頁應避免一次顯示所有資料。高頻操作放在前面，低頻管理功能放在次要區域。

## 表格與篩選

表格是 Lemon Platform 的主要資料呈現方式。每個表格都應明確顯示資料來源、更新時間、主要欄位與可執行操作。

篩選條件應放在資料上方或左側，並保留目前篩選狀態，避免使用者在查詢過程中迷失。

## 行動裝置考量

若平台需要在手機或平板上使用，應先支援查詢與確認類任務。大量編輯、財務核對與表格比較仍以桌面版為主要情境。
