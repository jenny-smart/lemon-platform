# 核心架構

## 核心概念

Lemon Platform 的核心架構以「中心化入口、模組化責任、可追溯資料流」為原則。平台不應把所有邏輯集中在單一功能頁，而是透過中心與模組分層管理。

## 核心能力

平台核心能力包含：

- 使用者入口與中心導覽。
- 模組頁面的共用結構。
- 既有系統與外部資料來源的整合。
- 狀態摘要、查詢、匯出與操作紀錄。
- AI Agent 可理解的任務與文件規則。

這些能力應優先穩定，再逐步增加新流程。

## 模組邊界

每個模組都應回答三個問題：

- 它負責哪一類業務任務。
- 它使用哪些資料來源。
- 它產生哪些輸出或狀態變更。

若一個功能同時影響多個中心，應指定主要擁有中心，其他中心只讀或引用結果。

## 資料責任

資料責任應與業務責任一致。例如訂單狀態應由訂單相關系統或 Service Center 負責，Finance Center 只使用付款與對帳所需的訂單資訊。

平台文件應記錄資料來源、更新頻率、手動修正方式與錯誤處理責任。

## 可觀測性

所有重要流程都應能追蹤輸入、處理、輸出與錯誤。即使初期只透過人工紀錄或文件，也應避免出現無法重現的處理結果。

Automation Center 未來應集中管理排程、執行紀錄與通知。

## 決策紀錄

重大架構決策應寫入文件，包含背景、選項、決策、影響與後續工作。未定案內容應標示 Draft 或 Pending，避免被誤認為已完成規格。
