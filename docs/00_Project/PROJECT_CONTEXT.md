# 專案背景

## 專案定位

Lemon Platform 是用來整合 Lemon 相關營運系統的主平台。它不是單一功能工具，而是把訂單、備忘、排班、薪資、財務、管理工具與自動化流程逐步收斂到同一個操作入口。

平台的核心目標是降低日常營運的切換成本，讓使用者可以用一致的資料理解、操作流程與介面邏輯完成工作。

## 現況脈絡

目前 Lemon 相關能力分散在多個系統與工具中，包括訂單處理、Memo 管理、薪資資料、工具中心與營運報表等。這些系統已經支撐實際工作，但在入口、資料流、權限與維護方式上仍需要統一。

Lemon Platform 的角色是作為整合層與未來主入口，逐步把既有能力整理成清楚的中心與模組。

## 使用者

主要使用者包含營運人員、客服人員、財務人員、管理者與維護工程師。不同角色關心的資料不同，但都需要穩定、可追溯、容易理解的操作流程。

平台設計應優先服務高頻任務，例如查詢訂單、確認狀態、整理待付款名單、檢查服務紀錄、處理異動與輸出報表。

## 成功標準

Lemon Platform 的成功不只取決於功能數量，也取決於是否讓日常工作更清楚、更少錯誤、更容易交接。

判斷標準包括：

- 常用任務能在單一入口完成。
- 重要資料來源與責任邊界清楚。
- 新成員能依文件快速理解系統。
- AI Agent 能依規則安全地協助維護與產出。
- 模組擴充時不破壞既有流程。

## 專案邊界

本專案包聚焦 Lemon Platform 的文件、架構與模組規劃，不直接新增應用程式碼，不搬移其他 repository 的內容，也不取代既有系統的原始維護文件。

當平台需要串接其他系統時，應以文件標明來源、責任與後續整合方式。

## 維護節奏

每次新增模組、調整資料流、改變 UI 基準或導入自動化流程時，都應檢查本文件是否需要更新。若實作與文件不同，應優先標示差異，避免讓文件看起來比系統更完整。
