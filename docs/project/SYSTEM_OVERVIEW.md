# 系統總覽

## 平台目標

Lemon Platform 目標是成為 Lemon 營運工作的統一入口。它負責整理多個既有系統的能力，提供一致的導覽、操作模式、資料檢視與任務流程。

平台本身應避免重複建立已存在且穩定的功能，而是優先以整合、封裝、導覽與標準化方式降低使用成本。

## 系統中心

平台規劃分為五個中心：

- Service Center：處理服務、客戶、地址、預約與服務狀態。
- Operations Center：處理營運追蹤、派工、異動與日常管理。
- Finance Center：處理付款、薪資、費用、對帳與財務輸出。
- Tools Center：集中既有工具、資料查詢與人工輔助作業。
- Automation Center：集中自動化任務、排程、通知與 AI 協作流程。

每個中心都應有明確責任，不應把所有功能塞進單一頁面。

## 資料流概念

資料流應以「來源清楚、轉換可追溯、輸出可重做」為原則。平台可以讀取既有系統資料、整理中間狀態、產生操作建議或報表，但不得模糊原始資料來源。

當資料來自外部表單、Google Sheet、既有 repository、Streamlit 工具或人工輸入時，文件應標明來源與更新方式。

## 使用流程

典型流程如下：

1. 使用者從平台入口選擇工作中心。
2. 中心頁面顯示高頻任務、狀態摘要與最近異動。
3. 使用者進入特定功能，例如訂單查詢、待付款整理或薪資檢查。
4. 系統顯示資料來源、處理結果與可執行動作。
5. 操作完成後留下可追蹤的結果或輸出。

## 權限與安全

平台應依角色控制可見資料與可執行動作。財務、薪資、客戶個資與內部備註屬於敏感資訊，文件與輸出不應包含未遮蔽的私密資料。

在權限尚未完整實作前，應以最小可見範圍與清楚人工檢查作為暫行原則。

## 整合策略

短期以文件化與入口整合為主，中期建立模組邊界與共用 UI，長期再把穩定流程整理為平台核心能力。

所有整合都應保留回退方式，避免 Lemon Platform 尚未成熟時影響既有營運。
