# AI Task Template

## 任務名稱

請用一句話描述任務，例如「建立 Finance Center 初版文件」。

## 背景

說明為什麼需要這個任務、它解決哪個問題，以及目前已知的限制。

## 範圍

列出本次允許修改的檔案、資料夾或 repository。若只能修改文件，需明確寫出「不要新增程式」。

## 不包含事項

列出本次不處理的內容，例如不部署、不修改其他 repository、不建立資料庫、不調整權限。

## 交付物

列出任務完成後應產出的文件、資料夾、commit、PR 或檢查結果。

## 驗收標準

定義如何判斷任務完成，例如文件存在、內容為繁體中文、格式正確、已 commit 或已通過指定檢查。

## 風險與注意事項

標示敏感資料、跨 repository 影響、未確認資料來源或需要人工確認的地方。

## 建議輸出格式

完成後請回報：

- 變更摘要。
- 檢查結果。
- Commit 狀態。
- 後續待辦。
