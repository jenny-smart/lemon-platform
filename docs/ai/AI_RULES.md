# AI Rules

## 基本規則

AI Agent 在 Lemon Platform 工作時，必須遵守任務邊界。使用者若要求只建立文件，AI 不得新增程式碼、修改應用邏輯或調整其他 repository。

所有輸出應使用繁體中文，除非使用者明確要求其他語言。

## Repository 規則

AI 應只修改使用者指定的 repository。若任務涉及其他 repository，必須先確認使用者是否授權。

在 Lemon Platform 中，Project Pack 相關變更應限於文件與資料夾，避免混入功能實作。

## 文件規則

文件應具備完整章節與可執行內容，不應只建立空標題。每份文件需標示狀態與版本，並清楚說明目前是 Draft 或已確認。

文件中不得放入密碼、Token、未遮蔽個資、內部憑證或不可公開的敏感資料。

## 程式規則

未經明確要求，不新增程式碼、不新增套件、不改設定、不啟動部署流程。

若未來任務需要程式變更，AI 應先讀取既有架構與測試方式，再進行最小範圍修改。

## 財務與個資規則

涉及付款、薪資、費用、客戶資料或地址時，AI 不得自行推測缺失欄位。若資料不完整，應標示待確認。

AI 不應在文件中寫入完整個資或敏感交易資訊。

## Commit 規則

若使用者要求 commit，AI 應在完成檢查後提交。Commit message 應遵守使用者指定文字；若未指定，應使用簡短且可追蹤的描述。

提交前應確認 Git status，避免把不屬於任務的變更一起提交。
