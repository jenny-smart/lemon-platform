# Sprints

## 目的

本文件用於規劃與紀錄 Lemon Platform 的 sprint。每個 sprint 應有清楚目標、範圍、交付物與驗收方式。

## Sprint 格式

每個 sprint 建議記錄：

- Sprint 名稱。
- 期間。
- 目標。
- 交付項目。
- 不包含事項。
- 驗收方式。
- 完成後紀錄。

## Sprint 0：Project Pack

目標是建立 Lemon Platform 的初始 Project Pack，包含專案背景、架構、模組、專案管理、AI 規則與基礎資料夾。

交付項目包括：

- `docs/00_Project/`
- `docs/01_Architecture/`
- `docs/02_Modules/`
- `docs/03_Project/`
- `docs/04_AI/`
- `assignments/`
- `templates/`
- `references/`

不包含事項：

- 不新增程式碼。
- 不修改其他 repository。
- 不部署任何服務。

## 驗收方式

Sprint 0 完成時，所有指定文件都應存在，內容使用繁體中文 Markdown，並在文件最後標示 Draft 與版本。

## 後續 Sprint 候選

後續 sprint 可聚焦 repository 盤點、UI 基準整理、中心功能清單、資料來源分類與 AI 工作流程建立。
