# Backlog

## 目的

本文件整理 Lemon Platform 後續待辦工作。Backlog 不是承諾清單，而是用來保存尚未排入 sprint 的需求、風險、研究項目與文件工作。

## 分類

Backlog 依下列類型整理：

- 文件：補齊專案、架構、模組、AI 與流程文件。
- 架構：確認平台分層、資料流、權限與整合方式。
- UI：整理基準介面、共用元件與版面規範。
- 模組：規劃五大中心的功能與責任。
- 整合：盤點既有系統、資料來源與外部工具。
- 自動化：定義排程、AI 協作與執行紀錄。

## 初始待辦

- 盤點 Lemon 相關 repository 與正式負責範圍。
- 確認 `tool-system` 中仍在使用的功能。
- 整理五大中心的第一版功能清單。
- 建立資料來源與敏感資料分類表。
- 定義平台首頁與中心頁的低保真版面。
- 補齊 AI Agent 的任務執行規則與審查流程。
- 決定短期是否只做文件與入口整合。

## 優先順序原則

優先處理能降低營運風險、減少人工重工、提升資料可追溯性的項目。純展示、低頻、未確認資料來源的項目應延後。

## Backlog 維護規則

新增待辦時應包含背景、預期結果、依賴與風險。完成或取消的項目應移到 sprint 紀錄或另行標記，不應長期留在未分類清單。
