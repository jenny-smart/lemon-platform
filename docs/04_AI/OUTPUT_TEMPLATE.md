# AI Output Template

## 目的

本文件定義 AI Agent 完成 Lemon Platform 任務後的回報格式，讓維護者能快速確認結果、風險與下一步。

## 建議回報格式

任務完成後，AI 應使用簡潔回報，包含：

- 完成項目。
- 修改範圍。
- 檢查方式。
- Commit 或 PR 狀態。
- 未完成或待確認事項。

## 文件任務回報

若任務只涉及文件，回報應說明建立或更新了哪些資料夾與文件，並確認沒有新增程式碼。

若有既有文件被更新，應說明更新原因。

## 程式任務回報

若未來任務涉及程式，回報應包含測試結果、主要檔案、行為變更與風險。若測試無法執行，需說明原因。

## Commit 回報

若已 commit，應回報 commit message 與提交狀態。若使用者要求推送或建立 PR，需另外回報遠端分支與 PR 連結。

## 風險回報

AI 不應隱藏不確定性。若資料來源未確認、權限不明、文件仍為 Draft 或任務邊界可能影響其他系統，應在回報中明確標示。
